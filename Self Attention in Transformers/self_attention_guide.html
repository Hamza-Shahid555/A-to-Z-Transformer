A comprehensive guide to understanding how Self-Attention turns static word embeddings into dynamic, context-aware representations.

01 — Overview

What is Self-Attention?

Self-Attention is a mechanism that transforms static word embeddings into dynamic, contextual embeddings. It is the foundational building block of the Transformer architecture, which in turn underpins virtually all modern Generative AI systems.

The core idea: a word like "bank" can mean a river bank or a financial institution. Self-Attention lets the model look at surrounding words to resolve that ambiguity — and encode the result into the word's representation.

02 — Mechanism

How It Works: The Basic Approach

In the simplest form, a word's new contextual embedding is computed as a weighted sum of all word embeddings. The weights are similarity scores — how much attention each word should pay to every other word.

The model computes a similarity score between every pair of words using the dot product. Higher scores mean greater relevance. Softmax normalizes scores to sum to 1 (attention weights). The final embedding for each word is a weighted combination of all embeddings.

Parallelization: All contextual embeddings can be computed simultaneously using matrix operations — making Self-Attention vastly more efficient than sequential models like RNNs.

03 — Limitation

The Problem: No Learnable Parameters

The basic approach has one critical flaw — no learnable weights. Without them, the model cannot adapt to a specific task. It produces the same generic contextual embedding regardless of whether we're translating, summarizing, or classifying.

Consider these examples in machine translation:

"Piece of cake" — should be translated idiomatically as "easy task," not literally as a dessert slice.
"Break a leg" — means "good luck" in English theatre, not a physical injury command.

A model without learnable parameters would fail on both. It needs task-specific embeddings that are shaped by training data — and for that, we need learnable weights.

04 — Query, Key & Value

Introducing Query, Key, and Value

The solution is to give each word embedding three distinct roles — and to learn separate transformations for each role using dedicated weight matrices.

Diagram 4 — The Three Roles of Every Word Embedding
🔍

Query (Q)

Asks: "Which other words are relevant to me?" Used to compute attention scores against Keys.

🗝️

Key (K)

Answers queries from other words: "How relevant am I to you?" Paired with Queries to produce scores.

📦

Value (V)

Contains the actual information to be aggregated in the weighted sum for the final output.

Forcing a single vector to simultaneously act as its own Query, Key, and Value is inefficient — like asking someone to simultaneously ask a question, answer it, and represent the information. Separating the roles allows each to be optimized for its specific function.

05 — Learnable Parameters

Weight Matrices Wq, Wk, Wv

Each word embedding e is projected into three vectors through matrix multiplication with three learned weight matrices:

The same Wq, Wk, Wv matrices are shared across all words in the sentence. They are initialized randomly and refined during training via backpropagation.
06 — The Formula

The Scaled Dot-Product Attention Formula

With Q, K, and V in place, the full attention calculation is:

Attention(Q, K, V) = Softmax( QKᵀ / √dk ) · V
QQuery matrix
KKey matrix
VValue matrix
√dkScaling factor

The scaling factor √dk (square root of the key dimension) prevents dot products from growing too large in magnitude, which would push Softmax into regions with very small gradients — making training unstable.

Why scale? For large embedding dimensions, dot products can become extremely large. Dividing by √dk keeps values in a range where Softmax produces meaningful, non-saturated gradients during backpropagation.

08 — Code

Self-Attention from Scratch (Python)

Here is a clean NumPy implementation of Scaled Dot-Product Self-Attention:

Python · NumPyimport numpy as np

# ── Helper ──────────────────────────────────────────────────
def softmax(x):
    """Row-wise softmax."""
    e = np.exp(x - np.max(x, axis=-1, keepdims=True))
    return e / e.sum(axis=-1, keepdims=True)

# ── Scaled Dot-Product Attention ────────────────────────────
def scaled_dot_product_attention(Q, K, V):
    """
    Args:
        Q : (seq_len, d_k)  — Query matrix
        K : (seq_len, d_k)  — Key matrix
        V : (seq_len, d_v)  — Value matrix
    Returns:
        output : (seq_len, d_v)  — Contextual embeddings
        weights: (seq_len, seq_len) — Attention map
    """
    d_k = Q.shape[-1]

    # Step 1: Compute raw attention scores
    scores = Q @ K.T / np.sqrt(d_k)          # (seq, seq)

    # Step 2: Normalize with Softmax
    weights = softmax(scores)                 # (seq, seq)

    # Step 3: Weighted sum of Values
    output = weights @ V                      # (seq, d_v)
    return output, weights

# ── Self-Attention Layer ─────────────────────────────────────
class SelfAttention:
    def __init__(self, embed_dim, d_k=None, d_v=None):
        self.d_k = d_k or embed_dim
        self.d_v = d_v or embed_dim

        # Learnable weight matrices (Xavier initialization)
        scale = np.sqrt(2.0 / (embed_dim + self.d_k))
        self.Wq = np.random.randn(embed_dim, self.d_k) * scale
        self.Wk = np.random.randn(embed_dim, self.d_k) * scale
        self.Wv = np.random.randn(embed_dim, self.d_v) * scale

    def forward(self, X):
        """
        X: (seq_len, embed_dim) — stacked word embeddings
        Returns contextual embeddings of same shape.
        """
        Q = X @ self.Wq     # (seq, d_k)
        K = X @ self.Wk     # (seq, d_k)
        V = X @ self.Wv     # (seq, d_v)

        output, weights = scaled_dot_product_attention(Q, K, V)
        return output, weights

# ── Demo ─────────────────────────────────────────────────────
if __name__ == "__main__":
    np.random.seed(42)

    # Sentence: ["Humans", "love", "smartphones"]
    # Each word → random embedding of dim 8
    seq_len, embed_dim = 3, 8
    X = np.random.randn(seq_len, embed_dim)

    attn = SelfAttention(embed_dim, d_k=4, d_v=4)
    output, weights = attn.forward(X)

    print("Input shape :", X.shape)      # (3, 8)
    print("Output shape:", output.shape)  # (3, 4)
    print("\nAttention weights (row = word, col = attends to):")
    print(np.round(weights, 3))
Output: weights[i][j] tells you how much word i attends to word j. Each row sums to 1.0 after Softmax.
09 — Code (PyTorch)

Self-Attention in PyTorch

The same mechanism with full gradient support for training via backpropagation:

Python · PyTorchimport torch
import torch.nn as nn
import torch.nn.functional as F

class SelfAttention(nn.Module):
    def __init__(self, embed_dim, d_k):
        super().__init__()
        # Learnable projection layers (no bias for simplicity)
        self.Wq = nn.Linear(embed_dim, d_k, bias=False)
        self.Wk = nn.Linear(embed_dim, d_k, bias=False)
        self.Wv = nn.Linear(embed_dim, d_k, bias=False)
        self.scale = d_k ** 0.5

    def forward(self, X):
        """X: (batch, seq_len, embed_dim)"""
        Q = self.Wq(X)                            # (B, seq, d_k)
        K = self.Wk(X)                            # (B, seq, d_k)
        V = self.Wv(X)                            # (B, seq, d_k)

        # Attention scores
        scores = Q @ K.transpose(-2, -1) / self.scale   # (B, seq, seq)
        weights = F.softmax(scores, dim=-1)              # (B, seq, seq)

        output = weights @ V                      # (B, seq, d_k)
        return output, weights

# ── Demo ─────────────────────────────────────────────────────
embed_dim, d_k = 16, 8
batch_size, seq_len = 2, 5

X = torch.randn(batch_size, seq_len, embed_dim)
attn = SelfAttention(embed_dim, d_k)
out, w = attn(X)

print(f"Input : {X.shape}")   # (2, 5, 16)
print(f"Output: {out.shape}")  # (2, 5, 8)
print(f"Attn  : {w.shape}")    # (2, 5, 5)

10 — Summary

Key Takeaways

Self-Attention is the engine that makes Transformers so powerful. Here's what to remember:

1. Context matters. The same word can mean different things in different sentences — Self-Attention captures this.
2. Parallelism. Unlike RNNs, all positions are processed simultaneously — enabling massive speedups.
3. Three roles. Q, K, V separation allows each projection to be specialized and learned independently.
4. Learnable. Wq, Wk, Wv are trained end-to-end via backpropagation on task data.
