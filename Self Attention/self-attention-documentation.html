01 / Introduction

What is Self-Attention?

Self-attention (also called intra-attention) is a mechanism that allows each element in a sequence to attend to — and gather information from — every other element in the same sequence. It was introduced in the landmark 2017 paper "Attention Is All You Need" by Vaswani et al. and forms the backbone of Transformer models.

Before self-attention, sequence modeling relied on RNNs and LSTMs which processed tokens one at a time, making it hard to capture long-range dependencies efficiently. Self-attention solves this by computing relationships between all pairs of positions simultaneously, regardless of how far apart they are.

🔑 Core Idea

Given a sequence of tokens, self-attention computes a weighted sum of all tokens for each position, where the weights reflect how relevant each token is to the current one. This lets every token "see" and incorporate information from all others in a single step.

Why It Matters

Self-attention has several critical advantages:

  • Parallelization — unlike RNNs, all positions are processed simultaneously
  • Long-range dependencies — token at position 1 can directly attend to position 1000 with no degradation
  • Interpretability — attention weights provide insight into what the model "looks at"
  • Expressiveness — flexible enough to learn complex relational patterns

The Library Analogy

Think of self-attention like a library lookup system:

📋 Query (Q)

Your search query — what you're looking for. "I want books about space exploration."

🔖 Key (K)

Index cards describing each book. "This book: space, science, rockets."

📚 Value (V)

The actual book content that gets retrieved once you find a match.

⚖️ Score

How well your query matches each key. High score → more of that book's content is retrieved.

In self-attention, every token simultaneously acts as a query (asking "what do I need?"), a key (advertising "what I contain"), and a value (providing "what I give when chosen").

03 / Mathematics

The Mathematics

Core Formula

The scaled dot-product attention formula is:

Attention(Q, K, V) = softmax( QKᵀ / √dk ) · V The fundamental self-attention equation

Where:

  • Q ∈ ℝn×dk — Query matrix (n = sequence length, dk = key dimension)
  • K ∈ ℝn×dk — Key matrix
  • V ∈ ℝn×dv — Value matrix (dv = value dimension)
  • QKᵀ ∈ ℝn×n — Raw attention score matrix (each cell = dot product of query and key)
  • √dk — Scaling factor to prevent vanishing gradients in softmax
  • Output ∈ ℝn×dv — Context-aware representations
⚡ Why √dₖ Scaling?

For large dk, dot products grow large in magnitude, pushing softmax into regions with very small gradients. Dividing by √dk keeps the variance of dot products around 1, assuming Q and K have unit-variance components. Without this, training becomes unstable.

Formally: if qᵢ, kⱼ ~ N(0,1), then qᵢ·kⱼ ~ N(0, dk), so dividing by √dk gives variance = 1.

Projection Matrices

In practice, Q, K, V are created by projecting the input embeddings X through learned weight matrices:

Q = X · WQ,   K = X · WK,   V = X · WV X ∈ ℝn×d_model is the input embedding; W matrices are learned

Since the same input X produces all three (Q, K, V), this is called self-attention — the sequence attends to itself. In cross-attention (used in encoder-decoder), Q comes from one sequence and K, V from another.

Attention Score Matrix

The raw score matrix S = QKᵀ / √dk has shape [n × n] — every token's query is compared to every token's key. After softmax row-wise, we get attention weights A where each row sums to 1. The output is then a weighted combination of values.

Sij = qi · kj / √dk  (raw score: how much token i attends to token j)

Aij = exp(Sij) / Σk exp(Sik)  (softmax along rows)

outputi = Σj Aij · vj  (weighted sum of values) Element-wise breakdown of the attention operation
04 / Step-by-Step Walkthrough

Self-Attention: Step by Step

Let's trace through the full computation for a sequence of 3 tokens: ["The", "cat", "sat"]

1

Embed the Tokens

Convert each token to a vector using an embedding lookup table. If d_model = 4, we get a matrix X ∈ ℝ3×4 where each row is a token embedding.

X = [[0.1, 0.4, 0.2, 0.9], ← "The"
     [0.5, 0.1, 0.8, 0.3], ← "cat"
     [0.2, 0.7, 0.4, 0.1]] ← "sat"
2

Compute Q, K, V via Linear Projections

Multiply X by three separate learned weight matrices WQ, WK, WV (each ∈ ℝd_model × d_k).

This projects each token into three roles: what it's asking for (Q), what it offers (K), and what it delivers (V).

3

Compute Raw Attention Scores

Compute the n×n score matrix: S = Q · Kᵀ. Each entry Sij measures alignment between token i's query and token j's key via dot product.

Scale by √dk: S_scaled = S / √d_k

4

Apply Softmax

Apply softmax along each row so scores sum to 1. Now each row of A is a probability distribution: "token i distributes its attention across all positions."

A ≈ [[0.5, 0.3, 0.2], ← "The" attends most to itself
     [0.1, 0.6, 0.3], ← "cat" attends strongly to itself
     [0.2, 0.4, 0.4]] ← "sat" attends to "cat" and itself
5

Compute Weighted Sum of Values

Multiply attention weights A by the value matrix V: Output = A · V.

Each output vector is a context-aware blend of all value vectors, weighted by attention. Token "sat" will now have a representation that has partially "absorbed" information from "cat".

6

Optional: Output Projection

In multi-head attention, the concatenated outputs are projected once more through WO to produce the final output. In single-head attention, the output matrix is used directly.

05 / Queries, Keys & Values

Q, K, V — Deep Dive

The Query-Key-Value decomposition is the conceptual heart of attention. Here's what each matrix actually represents:

🔍 Query (Q)

The "question" a token is asking about its context. WQ is trained to map token embeddings to a representation that captures "what kind of information do I need?"

Example: for the word "ate", its query might encode "I need: a subject (who ate?) and an object (what was eaten?)"

🔑 Key (K)

The "answer" that a token broadcasts about its own nature. WK maps embeddings to "what kind of information do I provide?"

When "dog" has a key that matches "ate"'s query well, "ate" will pay more attention to "dog".

📦 Value (V)

The actual content transferred. While keys determine how much to attend, values determine what information flows.

Values can encode different aspects than keys — a token might have a "noun" key but a value that encodes rich semantic content.

⚙️ Learned Separately

The three weight matrices WQ, WK, WV are entirely independent and learned from scratch via backpropagation.

This gives the model flexibility: it can learn separate "languages" for what to seek vs. what to offer.

💡 Key Insight: Why Separate Q and K?

Why not just compute dot products between token embeddings directly? Because raw embeddings are general-purpose, while Q and K can specialize. WQ and WK can learn that "verbs should attend to nouns" without polluting the raw embedding space. The separation also allows asymmetric attention: i attending to j doesn't require j to attend equally to i.

06 / Multi-Head Attention

Multi-Head Attention

A single attention head can only capture one "type" of relationship between tokens. Multi-head attention runs h parallel attention heads, each with its own Q, K, V projections, allowing the model to jointly attend to information from different representation subspaces.

MultiHead(Q, K, V) = Concat(head₁, ..., headₕ) · WO

where headᵢ = Attention(Q · WQi, K · WKi, V · WVi) h heads run in parallel; outputs concatenated and projected

Dimension Management

To keep computational cost constant regardless of h, each head uses reduced dimension:

  • d_model = 512, h = 8 → dk = dv = 512/8 = 64 per head
  • Total parameters for all heads ≈ same as one full-dimension head
  • After concatenation: 8 × 64 = 512, then projected by WO ∈ ℝ512×512

What Different Heads Learn

Research has found that different attention heads specialize in different patterns:

Head TypeWhat It CapturesExample
SyntacticSubject-verb, noun-adjective relationships"runs" → "dog"
CoreferencePronoun references"it" → "animal"
LocalAdjacent tokens, n-gram patternsNearby words
Long-rangeDistant dependencies across the sequenceBeginning ↔ end
PositionalRelative or absolute position"previous token"
📐 Dimensions in GPT-3

GPT-3 uses d_model = 12,288, h = 96 heads, d_k = 128 per head. That's 96 separate "perspectives" the model can simultaneously hold on any sequence. The sheer breadth of relational patterns that can be captured is a major reason for the model's power.

07 / Masked Attention

Masked Self-Attention

In autoregressive generation (language models like GPT), we must ensure that token at position i can only attend to positions ≤ i — not to future tokens. This is enforced via a causal mask.

How Masking Works

Before softmax, we add a mask matrix M to the score matrix:

S_masked = S + M

where Mij = 0 if j ≤ i, else -∞ -∞ → exp(-∞) = 0, so future positions get zero attention weight
Causal Mask — upper triangle = -∞ (blocked)
         pos0  pos1  pos2  pos3
pos0  [    0,   -∞,   -∞,   -∞  ]
pos1  [    0,    0,   -∞,   -∞  ]
pos2  [    0,    0,    0,   -∞  ]
pos3  [    0,    0,    0,    0  ]

Encoder (Bidirectional)

In BERT-style models, no masking is applied. Every token sees every other token. Full bidirectional context enables rich representations.

Decoder (Causal)

In GPT-style models, the causal mask is applied. Position i sees only positions 0…i. This enables autoregressive generation: generate token by token, left to right.

Cross-Attention (Encoder-Decoder)

In encoder-decoder models (e.g., original Transformer for translation), the decoder uses cross-attention: Q comes from the decoder's hidden states, but K and V come from the encoder's output. This allows the decoder to "look up" relevant information from the encoded source sequence.

08 / Positional Encoding

Positional Encoding

Self-attention is permutation invariant — if you shuffle the input tokens, you get the same output (also shuffled). This means it has no built-in sense of order. Positional encodings inject position information into the token embeddings.

Sinusoidal Positional Encoding (Original)

PE(pos, 2i) = sin(pos / 100002i/d_model)
PE(pos, 2i+1) = cos(pos / 100002i/d_model) pos = token position, i = dimension index

Properties of sinusoidal encodings:

  • Unique vector for every position
  • The distance between any two positions is consistent (PE(pos + k) can be linearly derived from PE(pos))
  • Extrapolates to longer sequences than seen during training

Learned Positional Embeddings

BERT and GPT use learned position embeddings — simple trainable vectors for each position, concatenated with token embeddings. Simpler but doesn't generalize to unseen sequence lengths.

Rotary Positional Encoding (RoPE)

Used in LLaMA, PaLM, and many modern models. Instead of adding positional info to embeddings, RoPE rotates Q and K vectors in a position-dependent way before computing dot products. The dot product between rotated vectors naturally encodes relative position:

RoPE(qᵢ, pos) = qᵢ · eiθ·pos (in complex notation) Encodes relative position through rotation; scales well to long contexts

ALiBi (Attention with Linear Biases)

Instead of adding position encodings to embeddings, ALiBi adds a linear bias to attention scores based on distance between query and key. Simple, effective, and generalizes well to long sequences.

Attention Variants

VariantKey IdeaUsed In

Multi-Head Attention h parallel heads, each with reduced d_k BERT, GPT, T5
Grouped-Query Attention (GQA) modern Multiple Q heads share K,V heads. Reduces KV cache size. LLaMA 2, Mistral
Multi-Query Attention (MQA) All Q heads share a single K and V head. Very fast inference. Falcon, PaLM
Sparse Attention Each token only attends to a subset of positions (local window, strided, etc.) BigBird, Longformer
Flash Attention efficiency Tiling-based implementation that avoids materializing full n×n attention matrix in HBM Most modern LLMs
Linear Attention Approximates softmax attention with O(n) complexity using kernel trick Performer, LinearTransformer
Cross-Attention Q from one sequence, K/V from another sequence Encoder-decoder models
Sliding Window Attention Each token attends only to ±w neighboring tokens Mistral, Claude (older)

Flash Attention — Why It Matters

Standard self-attention requires storing an n×n matrix in GPU memory (HBM). For n=65,536 tokens at float16, that's 8GB just for the attention matrix. Flash Attention computes the output in tiles without ever materializing the full matrix, reducing memory from O(n²) to O(n) while achieving the exact same result. It's now ubiquitous in production LLMs.

⚠️ The Quadratic Bottleneck

The O(n²) time and memory complexity is the main limitation of self-attention. For n = 1,000 tokens, the attention matrix is 1M elements. For n = 100,000 tokens (long documents), it's 10B elements — infeasible without approximations like Flash Attention, sparse attention, or linear attention methods.

KV Cache (Inference Optimization)

During autoregressive generation, at each new token we'd naively recompute K and V for all previous tokens. The KV cache caches computed K and V tensors so we only compute the new token's K and V at each step, reducing inference time from O(n²) per token to O(n) per token.

Without KV cache: for token n, compute attention over all n tokens → O(n²) total
With KV cache: store K1..n-1, V1..n-1, compute only Kn, Vn → O(n) total
11 / Implementation

Code: From Scratch

NumPy — Minimal Self-Attention

import numpy as np

def scaled_dot_product_attention(Q, K, V, mask=None):
    """
    Q: (n, d_k)  - Query matrix
    K: (n, d_k)  - Key matrix
    V: (n, d_v)  - Value matrix
    mask: (n, n) - Optional causal mask (True = masked out)
    """
    d_k = Q.shape[-1]
    
    # Step 1: Compute raw attention scores
    scores = Q @ K.T / np.sqrt(d_k)   # (n, n)
    
    # Step 2: Apply mask (causal masking)
    if mask is not None:
        scores = np.where(mask, -1e9, scores)
    
    # Step 3: Softmax along last axis (per query)
    exp_scores = np.exp(scores - scores.max(axis=-1, keepdims=True))
    attn_weights = exp_scores / exp_scores.sum(axis=-1, keepdims=True)  # (n, n)
    
    # Step 4: Weighted sum of values
    output = attn_weights @ V  # (n, d_v)
    
    return output, attn_weights


def self_attention_layer(X, W_Q, W_K, W_V):
    """Full self-attention layer with projection matrices."""
    Q = X @ W_Q   # (n, d_k)
    K = X @ W_K   # (n, d_k)
    V = X @ W_V   # (n, d_v)
    output, weights = scaled_dot_product_attention(Q, K, V)
    return output, weights

# Example
n, d_model, d_k, d_v = 5, 16, 8, 8
X    = np.random.randn(n, d_model)
W_Q  = np.random.randn(d_model, d_k) * 0.1
W_K  = np.random.randn(d_model, d_k) * 0.1
W_V  = np.random.randn(d_model, d_v) * 0.1

out, attn = self_attention_layer(X, W_Q, W_K, W_V)
print(f"Output shape: {out.shape}")    # (5, 8)
print(f"Attention shape: {attn.shape}") # (5, 5)

PyTorch — Multi-Head Attention

import torch
import torch.nn as nn
import torch.nn.functional as F
import math

class MultiHeadAttention(nn.Module):
    def __init__(self, d_model, num_heads):
        super().__init__()
        assert d_model % num_heads == 0
        
        self.d_model = d_model
        self.num_heads = num_heads
        self.d_k = d_model // num_heads
        
        # Projection matrices (all heads packed into one matrix)
        self.W_Q = nn.Linear(d_model, d_model, bias=False)
        self.W_K = nn.Linear(d_model, d_model, bias=False)
        self.W_V = nn.Linear(d_model, d_model, bias=False)
        self.W_O = nn.Linear(d_model, d_model, bias=False)

    def split_heads(self, x, batch_size):
        # (B, n, d_model) → (B, h, n, d_k)
        x = x.view(batch_size, -1, self.num_heads, self.d_k)
        return x.transpose(1, 2)

    def forward(self, x, mask=None):
        B, n, _ = x.shape
        
        # Project to Q, K, V and split into heads
        Q = self.split_heads(self.W_Q(x), B)  # (B, h, n, d_k)
        K = self.split_heads(self.W_K(x), B)
        V = self.split_heads(self.W_V(x), B)
        
        # Scaled dot-product attention (batched, multi-head)
        scores = Q @ K.transpose(-2, -1) / math.sqrt(self.d_k)  # (B,h,n,n)
        
        if mask is not None:
            scores = scores.masked_fill(mask == 0, float('-inf'))
        
        attn_weights = F.softmax(scores, dim=-1)               # (B,h,n,n)
        context = attn_weights @ V                                # (B,h,n,d_k)
        
        # Merge heads back: (B,h,n,d_k) → (B,n,d_model)
        context = context.transpose(1, 2).contiguous()
        context = context.view(B, n, self.d_model)
        
        # Output projection
        return self.W_O(context), attn_weights

# Usage
mha = MultiHeadAttention(d_model=512, num_heads=8)
x = torch.randn(2, 64, 512)  # batch=2, seq_len=64, d_model=512
out, attn = mha(x)
print(out.shape)   # torch.Size([2, 64, 512])

Using PyTorch's Built-in

# PyTorch has a highly optimized built-in MHA:
mha = nn.MultiheadAttention(embed_dim=512, num_heads=8, batch_first=True)

# Self-attention: query=key=value=x
x = torch.randn(2, 64, 512)
output, attn_weights = mha(x, x, x)

# Causal mask (for decoder/LM tasks):
causal_mask = nn.Transformer.generate_square_subsequent_mask(64)
output, _ = mha(x, x, x, attn_mask=causal_mask)
12 / In the Transformer

Self-Attention in the Transformer Architecture

Self-attention doesn't operate in isolation — it's embedded within a Transformer block that includes normalization, residual connections, and a feed-forward network.

Full Transformer Block

# One Transformer block (Pre-LayerNorm variant, used by most modern LLMs)
def transformer_block(x, attn, ff, ln1, ln2):
    # Self-attention sub-layer with residual connection
    x = x + attn(ln1(x))    # Pre-LN: normalize BEFORE attention
    
    # Feed-forward sub-layer with residual connection
    x = x + ff(ln2(x))      # FFN: two linear layers with activation
    
    return x

Key components around self-attention:

  • Layer Normalization — Stabilizes training. Pre-LN (normalize before) is now preferred over Post-LN (original paper)
  • Residual Connection — x + Attention(x) allows gradients to flow freely through deep networks
  • Feed-Forward Network — Two linear layers with a nonlinearity (GeLU/ReLU). Typically 4× the model dimension. Provides token-wise processing after attention aggregates across positions
💡 Attention vs. FFN: Division of Labor

Attention aggregates information across positions (communication step). The FFN processes each token's representation independently after aggregation (computation step). Some research suggests attention handles syntax/coreference while FFN stores factual knowledge (acting as a key-value memory).

Applications of Self-Attention

📝 NLP

Language models (GPT-4, LLaMA), BERT for classification, T5 for seq2seq, machine translation, summarization, question answering, code generation.

GPT-4V, Flamingo, LLaVA combine vision and language via cross-attention between image and text representations.

Graph attention networks (GAT), Tabular Transformers. Self-attention over graph nodes or table rows/columns.

14 / FAQ

Frequently Asked Questions

Q: Is attention always "soft" (differentiable)?

Standard attention is soft — it's a weighted average, so gradients flow through it. "Hard attention" picks exactly one position (argmax), but this is non-differentiable and requires reinforcement learning tricks. Soft attention is almost exclusively used in practice.

Q: Does self-attention replace positional information entirely?

No. Without positional encodings, self-attention is truly order-agnostic — "cat bites dog" and "dog bites cat" produce the same representations. Positional encodings (sinusoidal, learned, RoPE, ALiBi) are essential for the model to understand sequence order.

Q: What's the difference between self-attention and cross-attention?

In self-attention, Q, K, and V all come from the same sequence X. In cross-attention, Q comes from sequence A (e.g., decoder), but K and V come from sequence B (e.g., encoder output). Cross-attention lets two sequences interact.

Q: Why use attention instead of convolutions?

Convolutions have a fixed local receptive field (limited by kernel size), so modeling long-range dependencies requires many stacked layers. Self-attention has a global receptive field — every pair of positions interacts directly in O(1) layers. Convolutions are O(1) per element but need O(n/k) layers for range k; attention is O(n²) per layer but O(1) layers for any range.

Q: How does attention differ in encoder vs. decoder?

  • Encoder — Bidirectional self-attention (no mask). Every token sees all tokens. Used for understanding/embedding tasks.
  • Decoder — Causal (masked) self-attention + cross-attention. Used for generation.

Q: What is "attention head pruning"?

Research (Michel et al. 2019) showed that many attention heads can be removed at test time with minimal performance drop. Some heads are redundant; others are critical. This is used for model compression.

Q: How does attention handle variable-length sequences?

Self-attention naturally handles variable lengths because it operates on whatever tokens are present. Padding tokens are typically masked out (padded positions get -∞ scores before softmax) so they don't affect outputs of real tokens.
